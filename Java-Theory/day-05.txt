### Day 5: JDK, JRE, and JVM 

- **JDK (Java Development Kit)**  
  - Used by developers to build Java applications.  
  - JDK = JRE + Development Tools.  
  - Development tools include: compiler (`javac`), debugger, documentation generator (`javadoc`), and other utilities.  

- **JRE (Java Runtime Environment)**  
  - Required to run Java applications.  
  - JRE = JVM + Java Libraries.  
  - Examples of libraries: `java.util`, `java.io`, `java.lang`, `java.net`.  

- **JVM (Java Virtual Machine)**  
  - Executes Java bytecode.  
  - **Components of JVM:**  
    1. **Class Loader Subsystem**  
       - Loads `.class` files into memory.  
       - Handles loading, linking, and bytecode verification.  
    2. **Memory Areas**  
       - **Method Area:** Stores class metadata, static variables, runtime constant pool.  
       - **Heap Memory:** Stores objects and instance variables.  
        
         Student s = new Student(); // object created in heap
       
       - **Stack Memory:** Stores local variables, method calls, thread-specific data. Each thread has its own stack.  
       - **Program Counter (PC) Register:** Tracks current and next instruction.  
       - **Native Method Stack:** Used when Java interacts with native languages like C or C++.  
    3. **Execution Engine**  
       - **Interpreter:** Executes bytecode line by line (simpler but slower).  
       - **JIT Compiler:** Converts frequently used bytecode into native machine code for faster performance.  
    4. **Garbage Collector (GC)**  
       - Automatically removes unused objects from heap memory.  
       - Prevents memory leaks.  
     
       Student s = new Student();
       s = null; // eligible for garbage collection
