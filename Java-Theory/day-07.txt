### Day 7: IDE, Import, and Naming Conventions 

- **IDE (Integrated Development Environment)**  
  - Provides tools for Java development in one place.  
  - Features: run programs, compile code, debug applications, code suggestions (IntelliSense), project management.  
  - Popular IDEs: IntelliJ IDEA, Eclipse IDE, Apache NetBeans.  
  - Without an IDE, you’d manually type commands like:  
   
    javac Hello.java
    java Hello
  

- **Import Statement**  
  - Used to access classes from other packages.  
  - Example:  
   
    import java.util.Scanner;

- **Nested Packages**  
  - The dot (`.`) separates package levels.  
  - Example: `java.util.Scanner`  
    - `java` → top-level package  
    - `util` → subpackage  
    - `Scanner` → class  

- **Java Naming Conventions**  
  - **Package:** lowercase (e.g., `com.company.project`)  
  - **Class:** PascalCase (e.g., `StudentDetails`, `BankAccount`)  
  - **Methods/Variables:** camelCase (e.g., `calculateSalary()`, `studentName`)  
  - **Projects:**  
    - PascalCase (e.g., `CustomerManagement`)  
    - lowercase with hyphens (common in Maven projects, e.g., `customer-management`)  

- **Default Package**  
  - If no package is specified, the class belongs to the default package.  
  - Example:  
    ```java
    public class Test {
    }
    ```  
  - Limitation: classes in named packages **cannot** import classes from the default package.  
    ```java
    package com.example;
    import Test;   // ❌ Compilation error
    ```

---

### ✅ Day 7 Summary
- IDE simplifies development.  
- `import` is used to access classes from other packages.  
- Dot (`.`) represents package hierarchy.  
- Naming conventions:  
  - Package → lowercase  
  - Class → PascalCase  
  - Method/Variable → camelCase  
  -Projects → 
    - PascalCase
    - lowercase with hyphens
- No package declaration → Default Package.  
- Classes in the default package cannot be imported into named packages.
