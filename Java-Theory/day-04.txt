 Day 4: Compile Time vs Run Time

- **Compile Time**  
  - Source code is checked for syntax and type-related errors.  
  - Java compiler (`javac`) converts `.java` files into `.class` bytecode.  
  - Example command:  
    javac FileName.java
  
  - Examples of compile-time errors:  
    - Missing semicolon (`;`)  
    - Variable not declared  
    - Type mismatch  

- **Run Time**  
  - JVM executes the bytecode.  
  - Logic errors and exceptions may occur during execution.  
  - Example command:  
    java FileName
   
  - Examples of runtime errors:  
    - `ArithmeticException`  
    - `NullPointerException`  
    - `ArrayIndexOutOfBoundsException`  

- **Important Point**  
  - A Java class can be compiled without a `main()` method.  
  - The compiler only checks syntax correctness.  
  - Example:  
    
    class Test {
        int x = 10;
    }
  
  - This compiles successfully:  
    javac Test.java
   
  - But running it fails:  
    java Test

    Error:  
    Main method not found in class Test
