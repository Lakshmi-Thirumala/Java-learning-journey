## 📘 Day 19: Methods in Java,Instance vs Static Methods

### 🔹 What is a Method?
- A **method** is a block of code written once and reused many times.  
- Helps avoid repetitive code.  
- Can return values or just perform actions.  


### 🔹 Syntax
returnType methodName(parameters) {
    // statements
}

- **returnType** → can be primitive (int, boolean), non-primitive (String), or objects.  
- **methodName** → written in **camelCase** (e.g., `calculateSum`).  
- **parameters** → inputs to the method (optional).  


### 🔹 Example
public int addNumbers(int a, int b) {
    return a + b;
}

### 🔹 Calling a Method
- Writing a method is not enough — you must **call it**.  
- To call, you need an **object** of the class (unless the method is `static`).  

class Calculator {
    public int addNumbers(int a, int b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator(); // create object
        int result = calc.addNumbers(5, 10); // call method
        System.out.println("Result: " + result);
    }
}

### 🔹 Class and Object Syntax

ClassName objectName = new ClassName();
objectName.methodName();



### 🔹 Instance Method
- Also called **non-static method**.  
- Belongs to an **object** of the class.  
- To call it, you must first **create an object**.  
- Example:

  class Calculator {
      int add(int a, int b) {   // instance method
          return a + b;
      }
  }

  public class Main {
      public static void main(String[] args) {
          Calculator calc = new Calculator(); // object creation
          System.out.println(calc.add(5, 10)); // call instance method
      }
  }



### 🔹 Static Method
- Belongs to the **class itself**, not to any object.  
- Can be called directly using the **class name**.  
- Example:
 
  class Calculator {
      static int multiply(int a, int b) {   // static method
          return a * b;
      }
  }

  public class Main {
      public static void main(String[] args) {
          System.out.println(Calculator.multiply(5, 10)); // call static method
      }
  }
 

### 🔹 Why is `main` Static?
- The **JVM** calls the `main` method to start the program.  
- JVM does **not create objects** of your class automatically.  
- If `main` were non-static, JVM would need an object first — which it doesn’t do.  
- So `main` is made **static**, allowing JVM to call it directly by class name.


### ✅ Key Takeaway
- **Instance method** → needs an object.  
- **Static method** → belongs to the class, no object needed.  
- **Main method** → static, because JVM calls it directly without creating objects.

### ✅ Day 19 Summary
- Methods = reusable code blocks.  
- Syntax: `returnType methodName(parameters) {}`.  
- Call methods using objects.  
- Return types can be primitive, non-primitive, or objects.

## 🔹 Methods for Reusable Code
Instead of writing the same logic multiple times, you write it once inside a method and then **call it with different arguments**.

### Example

class Calculator {
    // Instance method
    int addNumbers(int a, int b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();

        // Reusing the same method with different values
        System.out.println("Sum 1: " + calc.addNumbers(5, 10));
        System.out.println("Sum 2: " + calc.addNumbers(20, 30));
        System.out.println("Sum 3: " + calc.addNumbers(100, 200));
    }
}

👉 Output:
Sum 1: 15
Sum 2: 50
Sum 3: 300


## 🔹 Why This Matters
- **Reusable logic** → write once, use many times.  
- **Different inputs** → same method works for multiple cases.  
- **Cleaner code** → avoids duplication and makes maintenance easier.  

## 🔹 Key Takeaway
- Methods are **reusable code blocks**.  
- You can pass **different values (parameters)** each time you call them.
