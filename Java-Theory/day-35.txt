Day 35– Constructors (Summary)

Constructor

A constructor is a special member of a class that is used to initialize objects.

It has the same name as the class.

It does not have a return type (not even void).

It is called automatically when an object is created.



---

Object Creation Syntax

ClassName objectName = new ClassName();

Example:

Student s1 = new Student();

Here, Student() is the constructor.


---

Default Constructor

If you do not write any constructor, the JVM/compiler provides a default constructor (also called a no-argument/unparameterized constructor).

Example:

class Student {
}

The compiler provides:

Student() {
}

automatically.


---

Parameterized Constructor

A constructor that accepts parameters is called a parameterized constructor.

Example:

class Student {

    String name;

    Student(String name) {
        this.name = name;
    }
}

Object creation:

Student s1 = new Student("Lakshmi");

This initializes the object in a single step.


---

Important Rule

If you write any constructor (for example, a parameterized constructor), the compiler does not generate the default constructor.

So if you need both:

Student() {
}

Student(String name) {
    this.name = name;
}

you must write the no-argument constructor yourself.


---

Constructor Overloading

A class can have multiple constructors.

This is called constructor overloading.

Rules:

✅ Number of parameters should be different.


Student()
Student(int id)
Student(int id, String name)

OR

✅ If the number of parameters is the same, their data types or order must be different.


Example:

Student(int id, String name)

Student(String name, int id)

These are valid because the parameter order is different.

❌ Invalid:

Student(int id)

Student(int rollNo)

The parameter list is the same, so this causes a compile-time error.


---

Constructor Selection

When an object is created, Java looks for the matching constructor.

Example:

Student s = new Student(10);

Java searches for:

Student(int id)

It does not call the default constructor.


---

Why Use a Parameterized Constructor?

Instead of assigning values one by one:

Student s = new Student();

s.name = "Lakshmi";
s.age = 24;

We can initialize everything at object creation:

Student s = new Student("Lakshmi", 24);

This makes the code cleaner and avoids forgetting to initialize fields.


---

Instance Variables

Instance variables are variables that:

Are declared inside the class.

Are declared outside methods, constructors, and blocks.

Belong to each object.


Example:

class Student {

    String name;
    int age;
}

Here, name and age are instance variables.

Features

They receive default values if not initialized.

int → 0

double → 0.0

boolean → false

Object reference → null


Memory for instance variables is allocated when an object is created.



---

Quick Revision

✅ Constructor = Special member used to initialize objects.

✅ Constructor name must be the same as the class name.

✅ Constructors have no return type.

✅ new ClassName() calls a constructor.

✅ If no constructor is written, the compiler provides a default (no-argument) constructor.

✅ If any constructor is written, the compiler does not provide the default constructor.

✅ Multiple constructors are allowed (constructor overloading).

✅ Overloading rules:

Different number of parameters, or

Same number but different parameter types/order.


✅ Parameterized constructors initialize objects at creation time.

✅ Instance variables are declared inside the class but outside methods.

✅ Instance variables get default values and memory is allocated when an object is created.
